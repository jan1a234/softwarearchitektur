Projektziel gemäß Aufgabenstellung

Offizielle Aufgabenstellung:

"In dieser Fallstudie werden Choreografie und Orchestrierung in Event-Driven Architectures verglichen. Zu diesem Zweck werden zwei funktional identische Prototypen entwickelt und evaluiert. Die Evaluation erfolgt sowohl qualitativ als auch quantitativ (z. B. anhand von Durchsatz oder Latenz)."

Zu untersuchende Aspekte (verpflichtend):

  • Änderbarkeit: Wie leicht lassen sich neue Services hinzufügen oder bestehende modifizieren?
  • Wartbarkeit: Wie gut ist das System zu verstehen und zu pflegen?
  • Verfügbarkeit: Wie verhält sich das System bei Ausfällen einzelner Komponenten?
  • Fehlertoleranz: Wie robust ist das System gegenüber Fehlern?
  • Erweiterbarkeit: Wie einfach können neue Features integriert werden?
Projektspezifische Ergänzungen:

Diese Liste soll projektspezifisch ergänzt werden. Zusätzliche Aspekte könnten sein: Performance, Skalierbarkeit, Monitoring & Observability, Konsistenz, Komplexität.

🎼 Choreografie

  • Dezentralisierte Kontrolle
  • Lose Kopplung
  • Hohe Autonomie der Services
  • Bessere Skalierbarkeit
  • Schwieriger zu überwachen
  • Komplexe Fehlerbehandlung

🎺 Orchestrierung

  • Zentrale Kontrolle
  • Einfaches Monitoring
  • Klare Workflow-Definition
  • Einfache Fehlerbehandlung
  • Single Point of Failure
  • Höhere Kopplung

🎯 Projektanforderungen

1. Zwei funktional identische Prototypen

  • Ein Prototyp mit Choreografie-Pattern
  • Ein Prototyp mit Orchestrierungs-Pattern
  • Beide müssen die exakt gleiche Funktionalität bieten
  • Verwendung realistischer Use Cases (z.B. E-Commerce Bestellprozess)

2. Qualitative Evaluation

  • Architekturanalyse und Bewertung
  • Code-Qualität und Struktur
  • Wartbarkeit und Verständlichkeit
  • Dokumentation der Design-Entscheidungen

3. Quantitative Evaluation

  • Durchsatz: Requests pro Sekunde
  • Latenz: Response-Zeiten (P50, P95, P99)
  • Ressourcenverbrauch: CPU, Memory
  • Skalierungsverhalten: Performance unter Last
  • Recovery-Zeit: Nach Service-Ausfällen

Wichtig: Die Ausarbeitung muss alle Bewertungskriterien erfüllen:
  • Vollständige Erfüllung der Zielvorgaben
  • Logischer Aufbau und klare Struktur
  • Gut recherchierte und dokumentierte Quellen
  • Sprachlich korrekt (Fachsprache)
  • Konsistentes Layout mit guten Abbildungen

💻 Implementierungsdetails

Use Case: E-Commerce Bestellprozess

Services zu implementieren:
  • Order Service - Bestellverwaltung
  • Payment Service - Zahlungsabwicklung
  • Inventory Service - Lagerbestand
  • Shipping Service - Versand
  • Notification Service - Benachrichtigungen

Choreografie-Implementierung

// Event: OrderPlaced { "orderId": "uuid", "customerId": "uuid", "items": [...], "totalAmount": 199.99, "timestamp": "2025-01-15T10:00:00Z" } // Services reagieren autonom auf Events // Payment Service: Subscribes to OrderPlaced // Inventory Service: Subscribes to PaymentProcessed // Shipping Service: Subscribes to InventoryReserved

Orchestrierungs-Implementierung

// BPMN Workflow in Camunda Start → Create Order → Process Payment → Reserve Inventory → Create Shipment → Send Notification → End // Orchestrator koordiniert alle Service-Aufrufe orchestrator.execute("orderWorkflow", orderData)

🎼 Choreografie Stack
